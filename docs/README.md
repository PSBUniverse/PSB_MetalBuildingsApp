# Docs Index (Beginner Learning Path)

Welcome to the PSBUniverse beginner documentation set.

If you are brand new, follow this order.

## Day 1: Understand The Big Picture

1. Read [PROJECT_OVERVIEW.md](PROJECT_OVERVIEW.md)
2. Read [KNOW_YOUR_SHELL.md](KNOW_YOUR_SHELL.md)
3. Read [ROUTING_GUIDE.md](ROUTING_GUIDE.md)

Goal for Day 1:

- Understand what the shell is
- Understand folder responsibilities
- Understand how URLs map to pages and APIs

## Day 2: Learn Data And File Flow

1. Read [CRUD_SUPABASE.md](CRUD_SUPABASE.md)
2. Read [DATABASE_TABLE_NAMING.md](DATABASE_TABLE_NAMING.md)
3. Read [FILE_USAGE_GUIDE.md](FILE_USAGE_GUIDE.md)
4. Read [STYLING_GUIDE.md](STYLING_GUIDE.md)

Goal for Day 2:

- Understand CRUD basics
- Understand table naming convention and table intent (`s`, `t`, `m`)
- Understand how frontend and backend files connect
- Understand color/font/button style standards

## Day 3: Build Your First Module

1. Read [HOW_TO_CREATE_MODULE.md](HOW_TO_CREATE_MODULE.md)
2. Build a test module using the checklist in that file
3. Verify route, API, and access mapping all work

Goal for Day 3:

- Create a working module skeleton
- Follow naming and folder conventions
- Avoid common beginner mistakes
