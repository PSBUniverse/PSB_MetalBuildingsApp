# 🧠 What a Lean-to ACTUALLY Is

Not:

> “extra roof stuck on the side”

It is:

> **A secondary structure attached to the main frame with its own slope, height, and supports**

---

# 🎯 Core Rules of Lean-to (Non-negotiable)

## 1. Attached to ONE SIDE ONLY

* left OR right OR front OR back

---

## 2. Has Its Own:

* width (projection)
* drop (lower height than main)
* roof slope (usually single slope)

---

## 3. MUST Connect to Main Structure

👉 This is where most implementations fail.

Lean-to:

* attaches to main wall **top edge**
* slopes downward away from building

---

# 🧱 Config Structure

```ts
leanTo: {
  enabled: boolean
  side: "left" | "right"
  width: number // how far it extends out
  drop: number // how much lower than main roof
  length: number // usually same as building
}
```

---

# ⚙️ Derived Values (CRITICAL)

```js
const attachHeight = config.height
const leanToOuterHeight = attachHeight - config.leanTo.drop

const leanToHalfWidth = config.leanTo.width
```

---

# 🧱 Build Flow

```js
buildBuilding(ctx) {
  group.add(buildMainStructure(ctx))

  if (ctx.leanTo.enabled) {
    group.add(buildLeanTo(ctx))
  }
}
```

---

# 🔧 buildLeanTo(ctx)

## 1. Determine Side Offset

```js
const sideSign = ctx.leanTo.side === "right" ? 1 : -1

const baseX = sideSign * ctx.halfWidth
```

---

## 2. Create Posts (Lean-to)

Posts run along length:

```js
for (const z of ctx.positions) {
  const post = createPost(
    baseX + sideSign * ctx.leanTo.width,
    z,
    leanToOuterHeight
  )

  group.add(post)
}
```

---

## 3. Create Beams

Two beams:

### Inner beam (attached to main)

```js
y = attachHeight
x = baseX
```

### Outer beam

```js
y = leanToOuterHeight
x = baseX + sideSign * leanTo.width
```

---

## 4. Roof (Single Slope)

This is KEY.

👉 One plane only.

```js
const slopeWidth = ctx.leanTo.width
const heightDiff = ctx.leanTo.drop

const slopeLength = Math.sqrt(
  slopeWidth**2 + heightDiff**2
)
```

---

## Create Roof Mesh

```js
const geo = new THREE.PlaneGeometry(ctx.length, slopeLength)

const roof = new THREE.Mesh(geo, roofMaterial)

roof.rotation.x = Math.PI / 2
roof.rotation.z = Math.atan(heightDiff / slopeWidth)
```

---

## Position Roof

```js
roof.position.set(
  baseX + sideSign * slopeWidth / 2,
  attachHeight - heightDiff / 2,
  0
)
```

---

# 🔥 IMPORTANT: Seam Alignment

Lean-to MUST:

* touch main roof/wall cleanly
* no gap
* no overlap

👉 If you see gap → your math is wrong
👉 If you see overlap → your positioning is wrong

---

# 🧱 Walls for Lean-to

Optional:

* open (just posts)
* partial panels
* fully enclosed

👉 Same wall system, but:

* height = leanToOuterHeight
* width = leanTo.width

---

# ⚠️ Common Mistakes (YOU WILL HIT THESE)

## ❌ Floating Lean-to

Not attached visually → looks fake

---

## ❌ Wrong slope direction

Roof slopes INTO building instead of away

---

## ❌ Same height as main

Then it’s not a lean-to anymore

---

## ❌ Misaligned grid

Posts don’t match main spacing

👉 instant “fake” look

---

# 🧠 Key Insight

Lean-to is NOT a separate system.

It is:

> **A dependent extension of the main structure**

---

# ⚙️ Dependency Rules

| Lean-to depends on | Why              |
| ------------------ | ---------------- |
| height             | attachment point |
| width              | position offset  |
| spacing            | post alignment   |
| roof type          | compatibility    |
