# 🧱 1. TWO-TONE SIDING (LIKE COMPETITOR)

### 🧠 Concept

Walls are split into:

* **bottom panel (wainscot)**
* **top panel**

👉 Not separate meshes per panel → just **2 stacked meshes per wall**

---

## 🔧 Implementation

### Step 1: Define config

```ts
siding: {
  colorTop: string
  colorBottom: string
  splitHeight: number // e.g. 3ft
}
```

---

## Step 2: Build wall (split)

```js
function buildWallSegment(width, height, length, config) {
  const group = new THREE.Group()

  const bottomHeight = config.siding.splitHeight
  const topHeight = height - bottomHeight

  // Bottom (wainscot)
  const bottom = new THREE.Mesh(
    new THREE.PlaneGeometry(width, bottomHeight),
    createWallMaterial(config.siding.colorBottom)
  )
  bottom.position.y = bottomHeight / 2

  // Top
  const top = new THREE.Mesh(
    new THREE.PlaneGeometry(width, topHeight),
    createWallMaterial(config.siding.colorTop)
  )
  top.position.y = bottomHeight + topHeight / 2

  group.add(bottom, top)

  return group
}
```

---

## 🔥 Important Detail

👉 Align textures across both meshes
or you’ll get visible seams.

# 🚪 2. DOOR / WINDOW CUTOUTS (REAL WAY)

## 💣 Brutal truth

If you “just place a door mesh on top”:

👉 It will ALWAYS look fake.

---

## ✅ Correct Approach: CSG (Boolean Subtraction)

Use:

* `three-csg-ts` (recommended)
* or `three-bvh-csg` (faster)

---

## 🔧 Example

### Step 1: Base wall

```js
const wall = new THREE.Mesh(
  new THREE.BoxGeometry(width, height, thickness),
  material
)
```

---

### Step 2: Door cutout

```js
const doorCut = new THREE.Mesh(
  new THREE.BoxGeometry(doorWidth, doorHeight, thickness + 0.1)
)
doorCut.position.set(x, doorHeight / 2, 0)
```

---

### Step 3: Subtract

```js
import { CSG } from 'three-csg-ts'

const wallWithHole = CSG.subtract(wall, doorCut)
```

---

### Step 4: Add actual door

```js
const door = new THREE.Mesh(
  new THREE.PlaneGeometry(doorWidth, doorHeight),
  doorMaterial
)

door.position.set(x, doorHeight / 2, thickness / 2 + 0.01)
```

---

## ⚠️ Rules

* Door must snap to grid or valid positions
* Door height ≤ wall height
* Always subtract BEFORE adding trim

# 🧱 3. TRIM SYSTEM (THIS IS WHY THEIR MODEL LOOKS CLEAN)

This is the missing polish.

---

## 🧠 Types of Trim You Need

| Type           | Purpose          |
| -------------- | ---------------- |
| Corner trim    | hides wall seams |
| Base trim      | bottom edge      |
| Roof edge trim | roof edges       |
| Ridge cap      | top of roof      |

---

## 🔧 Corner Trim

```js
function createCornerTrim(height) {
  return new THREE.Mesh(
    new THREE.BoxGeometry(0.2, height, 0.2),
    new THREE.MeshBasicMaterial({ color: "black" })
  )
}
```

Place at:

```js
(-halfWidth, height/2, -length/2)
(+halfWidth, height/2, -length/2)
...
```

---

## 🔧 Roof Edge Trim

Follow roof slope:

```js
const edge = new THREE.Mesh(
  new THREE.BoxGeometry(0.2, 0.2, length),
  trimMaterial
)

edge.rotation.z = roofAngle
edge.position.set(x, y, 0)
```

---

## 🔧 Ridge Cap

```js
new THREE.BoxGeometry(0.3, 0.2, length)
```

Place at:

```js
(0, height + rise, 0)
```

---

# 🧠 CRITICAL SYSTEM RULE

Order matters:

```
1. Build walls
2. Apply cutouts (doors/windows)
3. Add trims
4. Add doors/windows meshes
```

If you mess this order:
👉 overlaps, z-fighting, broken visuals

```js
buildWalls(ctx)
```

# ⚠️ Common Mistakes (YOU MUST AVOID)

* ❌ door floating on wall (no cut)
* ❌ trim clipping through roof
* ❌ panel textures misaligned
* ❌ using too many meshes for panels
